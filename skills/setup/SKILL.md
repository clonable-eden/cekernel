# /setup

Interactive setup for cekernel runtime environment. Creates the directory structure and writes a user profile to `~/.config/cekernel/envs/default.env`.

## Workflow

### Step 1: Ask CEKERNEL_VAR_DIR

Ask the user where to store cekernel runtime data.

```
cekernel のランタイムディレクトリを指定してください。
このディレクトリに IPC ファイル、ロック、ログ、スケジュールデータが格納されます。

デフォルト: ~/.local/var/cekernel
```

If the user accepts the default or provides a custom path, store it as `VAR_DIR`.

### Step 2: Ask CEKERNEL_BACKEND

Ask the user which backend to use.

```
バックエンドを選択してください:

1. headless (default) — ターミナル不要。バックグラウンドで実行
2. wezterm — WezTerm タブで Worker を可視化
3. tmux — tmux セッションで Worker を実行

選択 [1/2/3]:
```

Map the selection: 1 → `headless`, 2 → `wezterm`, 3 → `tmux`. Default is `headless`.

### Step 3: Create directory structure

Run the following commands using the chosen `VAR_DIR`:

```bash
mkdir -p "${VAR_DIR}/ipc"
mkdir -p "${VAR_DIR}/locks"
mkdir -p "${VAR_DIR}/logs"
mkdir -p "${VAR_DIR}/runners"
if [ ! -f "${VAR_DIR}/schedules.json" ]; then
  echo '[]' > "${VAR_DIR}/schedules.json"
fi
```

### Step 4: Write user profile

Write `~/.config/cekernel/envs/default.env`:

```bash
mkdir -p ~/.config/cekernel/envs
cat > ~/.config/cekernel/envs/default.env <<EOF
# cekernel user profile (generated by /setup)
CEKERNEL_VAR_DIR=${VAR_DIR}
CEKERNEL_BACKEND=${BACKEND}
EOF
```

### Step 5: Summary

Display the result and guide the user to further configuration:

```
セットアップ完了:
  ランタイムディレクトリ: ${VAR_DIR}
  バックエンド: ${BACKEND}
  ユーザープロファイル: ~/.config/cekernel/envs/default.env

その他の設定項目については envs/README.md を参照してください。
```

If the user selected `wezterm`, also mention:

```
WezTerm バックエンドを使用する場合は config/README.md を参照して
WezTerm プラグインをインストールしてください。
```

Read these files using the Read tool to provide the actual content path relative to the plugin directory.

## Important

- Do NOT run `sudo` at any point. The purpose of this skill is to avoid `sudo` by using user-local paths.
- If `~/.config/cekernel/envs/default.env` already exists, show the current content and ask the user whether to overwrite.
- Use `~` expansion correctly in bash (use `$HOME` in scripts).
